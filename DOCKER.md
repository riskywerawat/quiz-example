Dockerfile คืออะไร? รู้จักคำสั่งพื้นฐานและการใช้งานจริง 

เคยไหมครับ เวลาที่เราพัฒนาแอปพลิเคชันขึ้นมา แล้วต้องเอาไปรันบนเครื่องอื่น หรือส่งต่อให้เพื่อนร่วมทีม แต่กลับเจอปัญหา "เครื่องฉันรันได้ แต่ทำไมเครื่องนายรันไม่ได้ล่ะ?" 😫 ปัญหาสุดคลาสสิกนี้แก้ได้ด้วย Docker ครับ และหัวใจสำคัญของการสร้าง Docker Image ก็คือไฟล์ที่ชื่อว่า Dockerfile นั่นเอง 
Dockerfile เปรียบเสมือน "คู่มือ" หรือ "ใบสั่งยา" ที่บอก Docker ว่าจะต้องสร้าง Image ของเราขึ้นมาอย่างไรทีละขั้นตอน ตั้งแต่การเลือก OS พื้นฐาน, การติดตั้งโปรแกรมที่จำเป็น, การคัดลอกไฟล์โค้dของเราเข้าไป จนถึงการกำหนดคำสั่งสำหรับรันแอปพลิเคชันเมื่อ Container เริ่มทำงาน 
วันนี้เราจะมาทำความรู้จักกับคำสั่งพื้นฐานใน Dockerfile ที่ใช้กันบ่อยๆ พร้อมเปรียบเทียบคำสั่งที่ดูคล้ายกันแต่มีการใช้งานที่แตกต่างกันครับ 
 

คำสั่งพื้นฐานที่ควรรู้จัก 

FROM 
คำสั่งแรกสุดใน Dockerfile เสมอ ใช้สำหรับระบุ Base Image หรือ Image พื้นฐานที่เราจะนำมาต่อยอด เหมือนการเลือกว่าเราจะสร้างบ้านบนที่ดินแบบไหน จะเป็น Ubuntu, Alpine, หรือ Node.js รุ่นล่าสุดก็ได้ 

# เริ่มต้นจาก Official Image ของ Node.js เวอร์ชั่น 20 บน Alpine Linux 
FROM node:20-alpine  

WORKDIR 
คำสั่งนี้ใช้สำหรับกำหนด Working Directory หรือโฟลเดอร์หลักที่เราจะทำงานภายใน Container หลังจากคำสั่งนี้ถูกเรียกใช้ คำสั่งอื่นๆ ที่ตามมาอย่าง RUN, CMD, COPY, ADD จะทำงานโดยอ้างอิงจาก path นี้ 

# กำหนดให้เข้าไปทำงานในโฟลเดอร์ /app 
WORKDIR /app  

การใช้ WORKDIR ดีกว่าการใช้ "RUN cd /app" เพราะมันช่วยให้ Dockerfile ของเราอ่านง่ายและชัดเจนขึ้นเยอะเลยครับ 

RUN 
ใช้สำหรับรันคำสั่งต่างๆ บน Shell ของ Image ในระหว่างขั้นตอนการ build Image มักจะใช้เพื่อติดตั้ง dependencies หรือโปรแกรมที่จำเป็นสำหรับแอปพลิเคชันของเรา 

# อัปเดต package list และติดตั้ง curl 
RUN apk update && apk add curl 
 
# ติดตั้ง dependencies ของโปรเจกต์ Node.js 
RUN npm install  

COPY & ADD 
สองคำสั่งนี้ใช้สำหรับคัดลอกไฟล์จากเครื่องของเรา (Host) เข้าไปใน Image แต่มีความสามารถที่แตกต่างกันเล็กน้อย 
COPY: ทำหน้าที่ตรงไปตรงมา คือ คัดลอกไฟล์และโฟลเดอร์จากต้นทางไปยังปลายทางใน Image เท่านั้น ไม่มีอะไรพิเศษ 
ADD: มีความสามารถมากกว่า COPY เล็กน้อย คือนอกจากจะคัดลอกไฟล์ได้แล้ว ยังสามารถ แตกไฟล์ tar (unzip) โดยอัตโนมัติ และ ดึงไฟล์จาก URL มาไว้ใน Image ได้ด้วย 

แล้วควรใช้อันไหนดี? 
ตามคำแนะนำจาก Best Practices ของ Docker เอง แนะนำให้ใช้ COPY เป็นหลัก ครับ 🏆 เพราะมันทำงานตรงไปตรงมาและคาดเดาได้ง่ายกว่า ADD เหมาะสำหรับกรณีที่เราต้องการแค่คัดลอกไฟล์จริงๆ ส่วน ADD ให้เก็บไว้ใช้ในกรณีที่จำเป็นต้องใช้ความสามารถพิเศษของมัน เช่น การแตกไฟล์ tar โดยอัตโนมัติจริงๆ เพื่อให้ Dockerfile ของเราเข้าใจง่ายและปลอดภัยที่สุด 

# คัดลอกไฟล์ package.json และ package-lock.json เข้าไปใน /app 
COPY package*.json ./ 
 
# คัดลอกไฟล์ทั้งหมดในโปรเจกต์เข้าไปใน /app 
COPY . .  

EXPOSE 
ใช้สำหรับบอก Docker ว่า Container ของเราจะเปิด Port ไหนรอรับการเชื่อมต่อจากภายนอก เป็นเหมือนการทำเอกสารกำกับไว้เฉยๆ ไม่ได้เป็นการเปิด Port จริงๆ การเปิด Port จริงๆ จะต้องทำตอนรัน Container ด้วยคำสั่ง docker run โดยใช้แฟล็ก "-p" หรือ "-P" 

# บอกว่าแอปพลิเคชันของเราจะรันบน port 3000 
EXPOSE 3000  

CMD & ENTRYPOINT 
สองคำสั่งนี้ใช้กำหนดคำสั่งที่จะรันเมื่อ Container เริ่มทำงาน (Runtime) ซึ่งเป็นจุดที่หลายคนสับสนที่สุด มาดูกันครับว่ามันต่างกันอย่างไร 
CMD (Command): กำหนด Default Command ที่จะรันเมื่อ Container เริ่มทำงาน แต่ผู้ใช้งาน สามารถสั่งรันคำสั่งอื่นมาทับ (override) ได้ง่ายๆ ตอน docker run 
ENTRYPOINT (Entry Point): กำหนด คำสั่งหลัก ที่จะรันเสมอเมื่อ Container เริ่มทำงาน มันถูกออกแบบมาให้ Container ของเราทำงานเหมือนเป็นโปรแกรมหนึ่งโปรแกรม (Executable) ผู้ใช้ยังสามารถส่งค่าพารามิเตอร์เข้าไปต่อท้ายคำสั่งหลักได้ 
แล้วจะใช้คู่กันได้ไหม? 
ได้ครับ! และเป็นวิธีที่แนะนำด้วย โดยเราจะใช้ ENTRYPOINT เพื่อกำหนดคำสั่งหลักที่ตายตัว และใช้ CMD เพื่อกำหนด พารามิเตอร์เริ่มต้น (Default Parameter) 

ENTRYPOINT ["/bin/ping"] 
CMD ["localhost"]  

- ถ้าเรารัน "docker run <image>" เฉยๆ มันจะรันคำสั่ง "ping localhost" 
- ถ้าเรารัน "docker run <image>" มันจะรัน "ping google.com" (ค่า CMD ถูก override) 
วิธีนี้ทำให้ Image ของเรายืดหยุ่นและใช้งานง่ายมากๆ ครับ 

ตัวอย่าง Dockerfile ฉบับสมบูรณ์ 
ลองมาดูตัวอย่าง Dockerfile สำหรับแอปพลิเคชัน Node.js แบบง่ายๆ ที่รวมคำสั่งทั้งหมดเข้าด้วยกัน 

# Stage 1: Build the application 
FROM node:20-alpine AS builder 
WORKDIR /app 
COPY package*.json ./ 
RUN npm install 
COPY . . 
RUN npm run build 
 
# Stage 2: Create the final image 
FROM node:20-alpine 
WORKDIR /app 
COPY --from=builder /app/package*.json ./ 
COPY --from=builder /app/node_modules ./node_modules 
COPY --from=builder /app/dist ./dist 
 
EXPOSE 3000 
CMD ["node", "dist/main.js"]  

ในตัวอย่างนี้มีการใช้เทคนิค Multi-stage build เพื่อทำให้ Image สุดท้ายมีขนาดเล็กที่สุด โดยแยกขั้นตอนการ build ออกจากขั้นตอนการรันจริงครับ 
หวังว่าบทความนี้จะช่วยให้ทุกคนเข้าใจพื้นฐานของ Dockerfile และนำไปปรับใช้กับโปรเจกต์ของตัวเองได้ง่ายขึ้นนะครับ! 😊
